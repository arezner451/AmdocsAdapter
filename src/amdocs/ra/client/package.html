This is the part of the Amdocs J2EE Connector API that clients need. Follow these steps in a client to use an Amdocs EJB thru the connector:
1. Lookup "eis/AmdocsConnectonFactory" in AppServer's global naming context. This returns {@link amdocs.ra.client.AmdocsConnectionFactory amdocsConnectionFactory}.
2. Call {@link amdocs.ra.client.AmdocsConnectionFactory#createConnection(ConnectionSpecImpl connSpecImpl) createConnection} on {@link amdocs.ra.client.AmdocsConnectionFactory amdocsConnectionFactory} -got in previous step- with {@link amdocs.ra.share.ConnectionSpecImpl connectionSpecImpl} parameter.
3. Call {@link amdocs.ra.client.AmdocsConnection#getAmdocsRef getAmdocsRef} on {@link amdocs.ra.client.AmdocsConnection amdocsConnection} -got in previous step.
4. Cast the Amdocs EJB remote IF reference -got in previous step- to the desired Amdocs remote EJB IF type.
5. Call business methods on casted Amdocs remote EJB IF.
6. Call remove on casted Amdocs remote EJB IF -bear skipping.
7. Call {@link amdocs.ra.client.AmdocsConnection#close close} on {@link amdocs.ra.client.AmdocsConnection amdocsConnection}.
